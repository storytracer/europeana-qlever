# 1 Image preparation for IIIF

Optimal use of the IIIF standard in image repositories requires the presence of a **specific system environment** and an **appropriate image format**.

This means that the initial start consists of preparing the image content to be IIIF-compliant (green part of the graphic below). If you afterwards want to provide IIIF images to portals like Europeana, you also need to update and create metadata that is IIIF-compliant in a subsequent process (blue part of the graphic below).

# 2 Usind EDM data for IIIF Metadata preparation

The so-called [IIIF manifests](https://iiif.io/api/presentation/3.0/) provide the essential metadata for IIIF. The IIIF manifests are a very important unit for IIIF because they are packages that contain all the information about the digital objects (eg. images) themselves and the metadata. The manifests represent those digital objects (audio, image, text, video or 3D) and can be displayed online in a [IIIF-compatible viewer](https://iiif.io/get-started/iiif-viewers/).

Technically speaking, IIIF manifests are defined [JSON](https://en.wikipedia.org/wiki/JSON) metadata objects that can be used and reused in a platform-independent and flexible manner. To take full advantage of IIIF, you need a IIIF JSON manifest for each of your digital resources which will enable you to use the [IIIF Presentation API](https://iiif.io/api/presentation/3.0/).

Having to create the single manifests from scratch can be a time-consuming endeavor. Therefore, the EDM2IIIF Manifest Creation Tool was realised to provide a way to automatically generate IIIF manifests out of existing [EDM](https://pro.europeana.eu/page/edm-documentation) metadata.
