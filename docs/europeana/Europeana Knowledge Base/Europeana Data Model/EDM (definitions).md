# EDM (definitions)

This page presents the classes defined in EDM and the properties that may be associated with them.

EDM comprises two principal profiles:

- **External** – the set of basic EDM classes and properties to be applied by data providers.
- **Internal** – the profile of EDM as published by Europeana following the ingestion and processing of data.

These abbreviations are used throughout this specification to indicate the profile in which a given class or property is employed.

In addition, EDM includes specialised profiles developed for particular applications, which are available [EDM profiles](EDM%20profiles.md).

## Datatypes

The datatype for properties used by EDM are represented in this page as follows:

|  **Type**          |  **Description**                                                                                                                         |
|:-------------------|:-----------------------------------------------------------------------------------------------------------------------------------------|
| Literal            | A string value with an optional language tag (i.e. represented in RDF/XML using the xml:lang attribute with an IETF BCP 47 language tag) |
| Literal (Datatype) | A value that conforms to a particular data type (e.g. a floating point)                                                                  |
| Reference          | A reference to another resource using a unique identifier for that resource (i.e. a URI).                                                |
| Reference (Class)  | A reference to another resource of a specific class using a unique identifier for that resource (i.e. a URI).                            |

| **Definition** | Represents the cultural heritage objects that Europeana collects descriptions about.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **Usage**      | EDM enables the integration of data from multiple providers describing the same real-world resource, while maintaining explicit provenance for each data source. EDM extends the Object Re-use and Exchange (ORE) model by adopting its [proxy mechanism](https://www.openarchives.org/ore/1.0/datamodel#Proxy), which allows distinct provider-specific descriptions to coexist without loss of attribution. The use of `edm:ProvidedCHO` differs between data providers and aggregators (including Europeana). Data providers typically attach descriptive statements about the cultural heritage object (CHO) directly to the `edm:ProvidedCHO`. Aggregators, by contrast, rely primarily on the proxy mechanism to ensure that attribution to the original data source is preserved. In the internal representation of EDM at Europeana (EDM Internal), the `edm:ProvidedCHO` is restricted to a few properties (see below). In the external representation (EDM External), however, all properties applicable to `ore:Proxy` may also be asserted on `edm:ProvidedCHO`. |
